# Scenarios: Keego (SoleMate) Pro Overhaul

Feature: pro-overhaul · Date: 2026-05-31

## Scenarios (Success)

1. **shoe-first 런 (핵심 차별점)**: 홈에서 신발을 고르고 '러닝 시작' → 권한 허용 → GPS 워밍업(첫 3 fix 무시) 후 트래킹. 5km 달리고 저장 → 그 신발의 누적거리가 +5km 차감되고, 수명 링이 그만큼 줄며, 임계값 근접 시 교체 배지가 뜬다.

2. **정확도 낮은 GPS fix 무시**: 도심에서 accuracy 35m fix가 들어와도 거리에 반영되지 않는다(마지막 양호 위치 유지로 경로는 끊기지 않음). 총거리가 튀지 않는다 — `decideFix`/필터 단위 테스트로 검증.

3. **자동 일시정지/재개**: 러너가 신호등에서 멈춰 속도가 0.6 m/s 미만으로 6초 지속되면 자동 일시정지(시간/거리 누적 중단). 다시 1.0 m/s 초과로 2초 달리면 자동 재개. 깜빡임 없음 — `decideAutoPause` 단위 테스트로 검증.

4. **ProfileScreen 설정 실동작**: 단위를 km↔mi로 바꾸면 전 화면 표시가 즉시 환산되어 반영되고 AsyncStorage에 영속화. 주간 목표를 30km로 설정하면 홈/통계의 달성률이 실제 런 데이터로 갱신된다.

5. **코스 지도 + 내보내기**: 지난 런 상세에서 저장된 route가 svg 폴리라인으로 그려진다. '공유'를 누르면 거리/페이스/신발명이 담긴 한국어 텍스트 요약이 RN Share로 공유된다.

6. **목표 달성률 & 스트릭**: 주간 목표 대비 진행 링과 연속 러닝 스트릭이 실데이터로 표시되어 'keep going' 동기를 부여한다.

7. **빌드 무결성**: 모든 변경 후 `npx tsc --noEmit`, `npm run lint`, `npm test`가 통과하고 기존 신발/런 데이터가 보존된다.

## Anti-Scenarios (Failure to Prevent)

1. **거리 튐 회귀 금지**: 신호 불량(accuracy 큰 fix)이나 GPS 점프(좌표 순간이동, >12 m/s)가 총거리에 가산되어선 안 된다.

2. **자동 일시정지 데이터 손상 금지**: 일시정지/재개 전환 시 거리·시간이 음수가 되거나 유실되어선 안 된다(iron law). 일시정지 중 거리가 계속 쌓여서도 안 된다.

3. **권한 거부 크래시 금지**: 위치 권한 거부 시 앱이 죽거나 가비지 거리를 기록해선 안 된다 — 한국어 안내 + 설정 유도, 트래킹 차단.

4. **데이터 파괴 금지**: 심박 UI를 숨기더라도 기존 heart_rate 값을 삭제·마이그레이션하지 않는다. 신발/런 기록을 파괴적으로 변경하지 않는다.

5. **하드코딩 설정 잔존 금지**: ProfileScreen에 '주 5회'/'켜짐'/'킬로미터' 같은 동작 안 하는 하드코딩 값이 남아선 안 된다. `activeIdx={0}` 하드코딩이 남아 선택 신발이 무시돼선 안 된다.

6. **사진 업로드 차단 금지**: 사진 첨부 실패가 신발 저장 자체를 막아선 안 된다(사진 없이 저장, 재시도 제공).

7. **하드코딩 색/폰트 잔존 금지(디자인)**: 화면 코드에 theme 토큰 밖의 raw 색상/폰트가 남아선 안 된다. 브랜드 워드마크에 구 'SOLEMATE'가 남아선 안 된다.

8. **네이티브 무단 변경 금지**: android/ios 빌드 설정, KalmanFilter/calcDist를 테스트 없이 변경하거나 package-lock.json을 수동 편집해선 안 된다.

9. **시크릿 하드코딩 금지**: 키/시크릿을 코드에 박아선 안 된다.

## Slice 4 Scenarios (Phase 2 — 2026-06-03)

S4-1. **부상예방 경고**: 신발 마모도가 90%를 넘으면 홈/신발 상세에 "지금 교체해야 부상 없이 계속 달릴 수 있어요" 위험 경고가, 75~90%면 주의가 표시된다. 75% 미만은 차분(경고 없음).

S4-2. **신발 로테이션 추천**: 신발 2켤레 이상일 때, (런타입을 고르면) 그에 맞는 카테고리 + 가장 오래 쉰 신발이 추천 이유와 함께 추천된다. 보관 신발은 추천에서 빠진다. 1켤레면 추천 UI가 숨겨진다.

S4-3. **AddShoe 모델 브라우즈**: 브랜드를 고른 뒤 모델 입력칸을 (글자 없이) 클릭하면 그 브랜드 모델이 알파벳순으로 주르륵 떠 골라서 등록할 수 있고, 글자를 치면 기존처럼 필터되어 골라도 된다(두 방식 병행).

S4-4. **로컬 백업/복원**: ProfileScreen에서 내보내기로 신발/런/설정이 담긴 JSON을 공유/저장하고, 가져오기로 복원하면 동일 데이터가 그대로 살아난다. 손상된 파일은 거부되어 기존 데이터를 파괴하지 않는다.

S4-5. **공유 카드 이미지**: 런 상세에서 공유하면 거리/페이스/시간/신발/미니 코스맵이 담긴 카드 이미지가 생성되어 공유된다(svg toDataURL, 새 네이티브 의존 없음).

S4-6. **개인 챌린지**: "이번 달 100km" 거리 챌린지나 연속일 스트릭 챌린지를 만들면 실제 런 데이터로 진행률 링이 채워지고, 목표 도달 시 달성 뱃지가 뜬다.

S4-7. **러닝/기록 화면 정리**: 러닝 중 시간/평균페이스/케이던스 위의 아이콘이 사라지고, 기록탭 상단(기간+요약)이 콤팩트해져 최근 기록 리스트가 잘 보이며, 신발 카드의 중복 진행바가 사라지고 원형 링만 남는다.

### Slice 4 Anti-Scenarios

A4-1. **백업 데이터 파괴 금지**: 가져오기(import) 실패/손상 파일이 기존 신발·런 기록을 덮어쓰거나 지워선 안 된다.

A4-2. **로테이션 오추천 금지**: 보관(retired) 신발이나 존재하지 않는 신발이 추천되어선 안 되고, 1켤레 이하에서 추천이 떠선 안 된다.

A4-3. **네이티브 의존 추가 금지(공유카드)**: 공유 카드 때문에 react-native-view-shot 등 새 네이티브 모듈을 추가해선 안 된다(svg toDataURL로 해결).

A4-4. **부상경고 오작동 금지**: 마모도가 낮은(<75%) 신발에 위험/주의 경고가 떠선 안 된다(불필요한 불안 조장 금지).

## Slice 6 Scenarios (Phase 3 — 2026-06-03, 진짜 마모 모델 + 교체 예측)

S6-1. **체중 반영 실효 마모**: 같은 신발에 동일한 100km를 달려도, 체중 85kg 사용자의 실효 마모가 70kg 사용자보다 크게 계산된다(weightFactor). 체중 미설정 시 기준(1.0)으로 동작하며 차단되지 않는다 — `effectiveWearKm` 단위 테스트.

S6-2. **노면·페이스 보정**: 트레일 런은 로드 런보다, 레이스 페이스는 이지 런보다 같은 거리에서 실효 마모가 더 크게 잡힌다. 노면 미태그 런은 road로 처리된다 — 단위 테스트.

S6-3. **폼 경과시간 열화**: 거의 안 신은 오래된 신발도 시간 경과(ageWearKm)로 실효 마모가 누적되어, "저주행이어도 오래되면 교체 권장"이 성립한다 — 단위 테스트.

S6-4. **교체 예측**: 최근 28일 실효 주행률이 있는 신발 상세에 "이 페이스면 약 N주 후 교체 권장 · 예상 M월 D일"이 표시된다. 잔여가 0 이하면 "지금 교체 권장", 최근 주행이 없으면 "최근 기록 없음"으로 안내된다 — `forecastReplacement` 단위 테스트 + 신발상세 행동 테스트.

S6-5. **체중 설정·노면 태그 영속**: ProfileScreen에서 체중을 입력하면 AsyncStorage에 영속되어 다음 실행에도 마모 계산에 반영된다. 런 편집/수동입력에서 노면을 고르면 해당 런에 태그되어 실효 마모에 반영된다.

### Slice 6 Anti-Scenarios

A6-1. **데이터 파괴 금지**: 실효 마모 모델 도입이 기존 신발 `total_km`·런 `distance_km` 원본을 덮어쓰거나 변경해선 안 된다(실효 마모는 파생 표시값, 원본 불변).

A6-2. **음수·NaN·무한 금지**: 체중 미입력·노면 미태그·런 0개·target_km 0 등 엣지에서 실효 마모/예측이 NaN·음수·Infinity를 내거나 화면을 깨뜨려선 안 된다(graceful 기본값).

A6-3. **과장 경고 금지**: 휴리스틱 추정임을 무시하고 단정적("정확히 N일 남음")으로 표현해선 안 된다 — '약', '예상' 등 추정 톤 유지.

A6-4. **네이티브/백엔드 변경 금지**: Slice 6는 순수 lib + 기존 화면 UI + AsyncStorage만. 네이티브 파일·백엔드 스키마를 변경해선 안 된다.

## Slice 8 Scenarios (Phase 3 — 2026-06-09, 리텐션: 푸시 알림 + 주간/월간 리캡)

S8-1. **알림 결정 로직(순수)**: `lib/notifications.ts`의 `dueNotifications`가 상태(신발+forecast·주간목표 진척·마지막 런 날짜·알림설정)를 받아, 교체임박(Slice 6 forecast overdue/임박)·주간목표 진척(예: 금요일 목표 미달 시)·러닝 리마인더(설정 시각·해당일 미실행 시)에 대한 알림 의도 목록을 반환한다 — 단위 테스트로 각 타입 트리거/비트리거 검증.

S8-2. **알림 설정 토글·영속**: ProfileScreen 알림 설정에서 종류별(교체임박/주간목표/러닝리마인더) 토글과 리마인더 시각을 바꾸면 `notif_settings`(AsyncStorage json)에 영속되고, 다음 실행에 반영된다. 토글을 끄면 해당 종류 알림이 `dueNotifications`에서 제외된다(끄기 가능).

S8-3. **알림 권한 graceful**: POST_NOTIFICATIONS 권한을 거부해도 앱이 죽지 않고 나머지 기능(러닝·신발관리)이 정상 동작한다(비차단). 권한 거부 시 안내만 표시.

S8-4. **FCM 네이티브 통합·빌드 무결성**: `@react-native-firebase/messaging`가 기존 firebase(app/auth/firestore 24.0.0)·RN과 호환 버전으로 통합되어 `npx tsc --noEmit`/`npm run lint`/`npm test`가 green이고, 오케스트레이터의 `npx react-native run-android`(에뮬레이터) gradle 빌드가 성공한다. jest.setup에 messaging 모킹이 추가되어 단위/행동 테스트가 네이티브 없이 통과한다.

S8-5. **주간/월간 리캡(순수)**: `lib/recap.ts`가 런/신발을 받아 총거리·런수·평균페이스·최다착용 신발·신발별 마모·PR을 담은 주간/월간 요약을 반환한다 — 단위 테스트(합계·평균페이스·최다착용·빈 데이터 graceful).

S8-6. **리캡 보기·공유카드**: ProfileScreen/홈에서 리캡에 진입해 주간/월간 요약을 보고, 공유하면 요약이 담긴 카드 이미지가 생성되어 공유된다(svg toDataURL, Slice 4 공유카드 패턴 재사용 — 새 네이티브 의존 0).

### Slice 8 Anti-Scenarios

A8-1. **데이터 파괴 금지**: 푸시 알림·리캡 도입이 기존 신발/런/설정(기존 `settings_alerts` 인앱 배지 포함)을 파괴·덮어쓰지 않는다. `notif_settings`는 신규 키로 추가하고 기존 `AlertSettings`는 보존한다.

A8-2. **빌드 깨짐 머지 금지**: messaging 통합 후 gradle 빌드/메트로/tsc/lint/test를 green으로 만들 수 없으면 깨진 네이티브 상태를 main에 커밋하지 않는다 — 네이티브 변경을 되돌려 빌드를 원상복구하고 비호환 내용을 보고한다.

A8-3. **새 네이티브 의존 추가 금지(리캡 공유카드)**: 리캡 공유카드 때문에 react-native-view-shot 등 새 네이티브 모듈을 추가하지 않는다(svg toDataURL로 해결, Slice 4 패턴).

A8-4. **알림 스팸·시크릿 금지**: 끈 종류의 알림이 발생하거나 같은 알림이 중복 발생해선 안 된다. FCM 서버키/서비스계정 등 시크릿을 코드에 박지 않는다(google-services는 레포 밖).

A8-5. **NaN·음수 금지(리캡)**: 런 0개·신발 0개·거리 0 등 엣지에서 리캡이 NaN·음수·Infinity를 내거나 화면을 깨뜨려선 안 된다(graceful 기본값).
