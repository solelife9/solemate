# Keego 전수 갭 감사 (2026-05-31, 능동 audit)

4개 전문 렌즈(엔진 신뢰성 / 기능 완성도 / UX·디자인 / 코드건전성)로 실제 코드+8개 스크린샷을 감사. Slice 반영용 source.

## 🔴 P0 — 출시 전 반드시 (대부분 Slice 1)

### 엔진 신뢰성 (running app 생사 직결)
1. **백그라운드 트래킹 부재** — AndroidManifest에 FOREGROUND_SERVICE/LOCATION 권한·서비스 없음. 화면 꺼지거나 앱 백그라운드 시 GPS·가속도계 콜백 정지 → 주머니에 넣고 달리면 거리 일부만 기록. **러닝앱 최고 우선 결함.** (Slice 1)
2. **크래시/킬 시 런 전손** — 모든 런 상태가 ref(in-memory)에만, 저장 버튼 전까지 미영속. OS가 메모리 부족으로 앱 종료(특히 화면off 장거리) 시 런 전체 소실. → 수 초마다 진행중 스냅샷 AsyncStorage 저장 + 재실행 시 복구. (Slice 1)
3. **저장 네트워크 실패 시 완주 런 소실** — addRun이 POST 성공 후에만 route 로컬 저장, 실패 시 화면 teardown으로 전손. → 로컬 우선 저장 + 미동기 큐. (Slice 1)
4. **pausedMs 계산 버그** — auto-pause 진입부 미설정 상태에서 resume 수식(App.tsx:440) 도달 시 pausedMs 폭주 → 시간/페이스 음수·오류. auto-pause 배선 시 guard 필수. (Slice 1)
5. **거리 하한 게이트 3m가 일반 페이스 과소계상** — d>0.003km(3m) 미만 구간 폐기(App.tsx:478). 1s 간격 3m/s=10.8km/h, 그보다 느린 대다수 러너는 fix간 델타가 버려져 **거리 체계적 과소집계**. → 하한 1m로 낮추거나 정확도 기반 게이트. (Slice 1)
6. **accuracy 미필터 → 유령거리** — 모든 fix를 Kalman에 투입(App.tsx:463), 콜드픽스/도심 50m+ fix가 3~300m 델타로 거리 가산. → accuracy>20m 거부 + 워밍업. (Slice 1)

### 데이터 무결성 (코드건전성)
7. **신발 total_km 미영속, 매 렌더 재계산** — addRun이 shoe PATCH 안 함; "used"=runs.filter().reduce()+start_km 파생(App.tsx:185·197 중복). 런 목록 일부 유실/페이징 시 신발 마일리지 조용히 감소, start_km 이중계산 위험. → 단일 순수함수 shoeUsedKm/shoeHealth(shoe,runs)로 통일·테스트, 서버 truth 영속(Slice2). **최대 데이터 무결성 갭.** (Slice 1→2)
8. **순수 로직 전부 App.tsx에 갇힘(미export)** — calcDist/fmt*/stats/parseShoeName 등 import·테스트 불가. KalmanFilter/auto-pause/cadence도 클로저에 묶임. → lib/geo·format·stats·shoe.ts 추출 + export, jest.setup 네이티브 모킹. harness 60% 커버리지 전제. (Slice 1)

### UX (신뢰 훼손)
9. **로딩/에러 상태 부재** — initUser fetch 실패가 빈-신규 사용자와 동일 렌더(콜드백엔드 30s+). → skeleton/에러카드(재시도)/empty 3상태. (Slice 2)
10. **디자인 스케일 부재** — theme.ts에 색·폰트만, spacing/radius/type 토큰 없음 → 패딩·폰트사이즈·radius 제각각(폰트 30종). 리뉴얼의 근본 선행. (Slice 1 토큰 → Slice 3 적용)

## 🟠 P1 — 중요 (차별점 심화 + table-stakes)

### shoe-first 차별점 심화 (PM 렌즈 — Keego의 해자)
- **은퇴(retire)/보관 기능** — 현재 deleteShoe가 런까지 cascade 삭제(ShoesScreen:43). 내구도 앱이 신발 정리하면 기록 소실 = 자기모순. retire(기록 보존, 선택목록 숨김) 분리. (Slice 1)
- **검증 수명 기반 비례 교체 경고** — 현 하드코딩 100km 잔여 경고. 400km 레이서와 800km 트레이너가 같은 버퍼는 틀림. 카테고리 수명(시드DB) 기반 % 티어(주의 75%/교체 90%) + 3단계 상태(양호/주의/교체). (Slice 1)
- **"오늘은 이 신발" 추천** — 홈이 "어떤 신발로?"라 묻지만 답을 안 줌. 최근 미착용/회복 휴식 기반 로테이션 추천 칩 → 차별점을 행동으로. (Slice 1~2)
- **신발 상세에서 바로 "이 신발로 달리기" CTA** — 신발 탭/상세에 런 시작 동선 없음(홈으로 돌아가야). shoe-first 최대 IA 개선. (Slice 2)
- **per-shoe 마지막 착용/타임라인, cost-per-km, 구매일** — 로테이션 데이터 의존 + 재참여 훅 + 차별 플렉스. (Slice 2)
- **개별 런 편집/삭제, 수동 런 입력** — 현재 불가. GPS 오측정이 신발 수명을 영구 왜곡(내구도 앱의 핵심 약속 훼손). 앱 외 주행 보정. (Slice 2)
- **PR(개인기록), 런 타입(이지/템포/롱)** — table-stakes + 런타입은 멀티슈즈 로테이션의 다리. (Slice 2~3)

### 엔진/UX P1
- 권한 거부/주행중 회수 graceful 처리, **iOS 권한요청 자체 없음**. GPS 死구간 시간만 누적. 케이던스 60s 윈도우 초기 과소·라벨 모호(2족 vs 총 spm). 200점 route 단순 데시메이션(RDP 권장). 주간 통계 UTC/로컬 날짜 불일치(심야 런 오분류). (Slice 1~2)
- 접근성: T3 #8E8E93 소형 텍스트 WCAG AA 미달, 색상 단독 상태표시(색맹), SR 라벨 전무, Dynamic Type 미지원, 터치타깃 44pt 미만(삭제 38px). (Slice 2~3)
- 숫자+단위 cramping("0.0km") 전역 → <Metric value unit/> 프리미티브. RunActive 글랜서블 위계(거리>시간>케이던스). 런 버리기 무확인(완주 런 1탭 폐기). 빈 차트가 깨진 듯 그려짐. safeArea 대신 paddingTop:60 하드코딩. (Slice 2~3)

## 🟡 P2 — 폴리시/정리
- 죽은 deps(@react-navigation/*, react-native-screens, rxjs) 제거 또는 채택 결정. ErrorBoundary 없음(throw→백스크린). console.log/빈 catch로 실패 은폐. any 남용(백엔드 경계 무타입) → BackendShoe/Run 타입. vector-icons any. substr deprecated. 단위 표기 불일치(1KM/km). 탭 신발 아이콘=발자국(중복 은유). press 피드백 누락. 프로필 이름 하드코딩 '러너'·아바타 가짜 어포던스. AddShoe 기본 max=500(모델엔 없는 값)·권장 배지 없음. (Slice 2~3)
