# Keego 탑티어 로드맵 ("전부 다")

사용자 요청: 탑티어 러닝앱 수준으로 완성도 올리기. 아래 전 항목을 순서대로 진행한다.
순수 코드(에뮬레이터에서 바로 검증) → 네이티브 스캐폴딩(실기기 빌드 필요) → 기기/콘솔 순.

상태: ☐ 예정 · ◐ 진행 · ☑ 완료(테스트+푸시)

---

## Phase 1 — 차별점 심화(신발 인텔리전스) + 분석 깊이 · **순수 코드**
가장 레버리지 높고 네이티브 의존 없음. 에뮬레이터에서 즉시 검증.

- ☑ **1-1 마모 예측 투명성** — 상세 교체예상 카드에 '왜 N주' 근거 한 줄 + 정확도(높음/낮음)
  칩. lib/wearView.forecastBasisKo/forecastConfidenceKo. 에뮬 검증 완료.
- ✖ **1-2 cost-per-km** — **제외(사용자 결정, 2026-06-12: "필요 없을 것 같다")**. 진행 안 함.
- ☑ **1-3 개인기록(PR)** — 기록 탭에 최장거리·최고페이스·최장시간·최장스트릭 2x2 카드.
  lib/records.personalRecords(전체 런 순수 파생). 에뮬 검증 완료.
- ☐ **1-4 신발 로테이션 인사이트 확장** — 휴식 분산/추천 근거 문구 강화(rotation 재사용).
  ('부상 막아준 횟수' 같은 추정 지표는 신뢰성 위해 보류).

## Phase 2 — 신뢰성/관측성 · **네이티브 스캐폴딩 + 실기기**
출시 안정성. 실사용자 크래시/이탈을 볼 수 있어야 개선 가능.

- ☐ **2-1 Crashlytics** — Firebase 이미 사용 → @react-native-firebase/crashlytics 추가. (네이티브 빌드)
- ☐ **2-2 분석 이벤트** — 핵심 퍼널(온보딩→첫신발→첫런→리텐션) 이벤트. (Firebase Analytics)
- ☐ **2-3 백그라운드 트래킹 무결성** — 화면off/장시간/저배터리 GPS 무손실 실기기 검증.
- ☐ **2-4 실기기 FCM 푸시 검증** — 리텐션 알림 실제 수신 확인(기록상 미완).
- ☐ **2-5 트레드밀/실내 모드** — GPS 끊김 시 케이던스/수동 거리 폴백.

## Phase 3 — 건강·데이터 연동 · **네이티브 + 콘솔**
진지한 러너 확보 + 신발 자동 거리 합산 시너지.

- ☐ **3-1 Health Connect / Apple Health 동기화** — 런/거리 양방향. (네이티브)
- ☐ **3-2 심박(BLE) + HR존 + 트레이닝 부하/회복** — Slice 5 계획분. (네이티브)

## Phase 4 — 완성도/성장 · **혼합**
무료앱과 탑티어를 가르는 "느낌" + 글로벌/수익.

- ☐ **4-1 애니메이션/햅틱/전환** — reanimated + 햅틱. 빈/에러/오프라인 상태.
- ☐ **4-2 i18n(영어)** — 현재 한국어 전용 → 글로벌.
- ☐ **4-3 프리미엄 티어 + 어필리에이트 강화** — 고급 마모분석/무제한 신발 + 신발 추천 링크 수익.
- ☐ **4-4 ASO/리뷰 유도/스토어 자산**.
