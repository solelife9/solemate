## 0. 먼저 해결한 근본 문제 (사용자가 겪은 "옛날 화면 / 조금만 바뀜"의 진짜 원인)

| 증상 | 원인 | 조치 | 커밋 |
|---|---|---|---|
| "디자인 바꿔도 옛날 화면" | APK에 **6/3 옛날 JS 번들**(`android/app/src/main/assets/index.android.bundle`)이 박혀 있어, Metro 불안정 시 앱이 그 옛날 번들로 폴백 | 옛 번들 제거 + 디버그 재빌드 → 항상 Metro(현재 코드) 로드. **핫리로드 정상 작동 확인** | (빌드 산출물, git 무관) |
| "온보딩 사진이 안 보임" | `LinearGrad`(react-native-svg)가 `stopColor` 의 rgba **알파를 무시** → 히어로 페이드가 불투명 검정 판이 되어 배경 사진을 가림 | `splitStopColor` 로 rgba→색+`stopOpacity` 분리. Welcome 히어로 사진 정상 표시 | `0bb416f` |

→ 이제 JS 저장 시 에뮬레이터에 즉시 반영됨. 디자인 작업의 전제 조건이 갖춰짐.

## 1. 완료 — 홈 화면 (커밋 `ca13378`, 테스트 `04ab985`)

**바뀐 것 (🎨):** '오늘의 신발' 섹션을 디자인의 **풀폭 스와이프 캐러셀**로 재구성.
- 히어로 1장 + 작은 피커 → **풀폭 카드 가로 스냅 캐러셀** + 페이지 도트(오렌지 활성) + "내 러닝화 N켤레 · 좌우로 넘겨보세요" 힌트.
- 카드에 **컨디션 한 줄(reason)** 추가: "오늘 데일리 러닝에 가장 좋은 컨디션이에요" 등(keep-going 보이스, 목업 reason 정합).
- 활성 카드 = 히어로(`home-hero`) + 교체예측 노출. 카드 탭 → 신발 상세. 러닝시작은 캐러셀 아래 단일 CTA(활성 신발 기준).

**유지한 것 (✅, 현재가 더 낫거나 동급):**
- 인사말 "오늘은 어떤 신발로 달려볼까요?" (이름 없음) — **사용자 명시 결정** (디자인의 "지훈님, …"은 채택 안 함).
- 상단 Keego 워드마크 + 신발추가 / 주간목표 통계(거리·러닝·**연속**) / 로테이션 추천 / 다음 러닝화 추천(어필리에이트) / 하단 탭바.

**아직 안 한 것 (디자인엔 있으나 데이터 배선 필요 — 다음 차례):**
- **현재 상태 인사이트 카드**(사용거리/교체예상/추천용도+태그) → 추천용도·태그는 신발 카테고리 매핑 필요.
- **최근 러닝** 리스트(홈) → App.tsx가 `runs`(UI 변환)를 Home에 내려줘야 함.

스크린샷: 작업 전 `%TEMP%/keego-home-before2.png`, 작업 후 `%TEMP%/keego-home-after2.png`.

## 2. 남은 핵심 화면 — 분석 & 권고 (다음 세션에서 같은 루프로 진행)

각 화면: 현재 컴포넌트 읽기 → 디자인 코드/사진과 "변경/유지/판단" 표 → 다시 칠하기(기능 보존) → 핫리로드 비교 → 테스트 green → 커밋.

### 신발 탭 (`ShoesScreen.rn.tsx`, 디자인 `07-shoes.png` / `ShoesScreen.js`)
- 디자인: "신발" 큰 제목 + "N켤레와 함께 Nkm 달렸어요" + 카드 스택(브랜드/모델/상태/**용도 한 줄**/**큰 누적 km**/교체까지/**가로 라벨바**) + 점선 "신발 추가".
- 현재: Ring(%) 카드 + 평균페이스 + 바로 달리기 버튼(▶) — **기능이 더 풍부**.
- 권고: 헤더 문구(N켤레와 함께 Nkm) 추가, 카드에 **용도 한 줄** + **가로 라벨바(used/max)** 도입은 디자인이 더 깔끔. Ring vs 바는 판단 — Ring+평균페이스+▶는 유지 가치 있음(현재 강점). 점선 추가 버튼 채택.

### 신발 상세 (`ShoesScreen.rn.tsx` 내 `ShoeDetail`, 디자인 `09-shoe-detail.png` / `ShoeDetailScreen.js`)
- 디자인: 뒤로 + StatusPill(lg) + 모델 + 잔여수명 + 통계(평균페이스 등) + 교체예상 + 인사이트 + 이 신발 최근 기록 + "이 신발로 러닝 시작".
- 현재: 매우 풍부(수명 보정·은퇴/복원·2×2 통계·기록·다음러닝화·부상배너).
- 권고: 디자인의 **잔여수명 강조 + 통계 레이아웃 + 인사이트 카드** 시각 정렬, 현재의 부가 기능은 전부 유지.

### 기록 (`HistoryScreen.rn.tsx`, 디자인 `10-records.png` / `RecordsScreen.js`)
- 디자인: "기록" + **세그먼트(주/월/년/전체)** + 요약(거리·횟수·평균페이스·총시간) + **주간 거리 추이 차트** + 러닝 리스트.
- 권고: 세그먼트 + 요약 4지표 + 추이 차트(react-native-svg) 도입. 현재 리스트/필터 기능 보존.

### 마이 (`ProfileScreen.rn.tsx`, 디자인 `12-my.png` / `MyScreen.js`)
- 디자인: "마이" + 프로필(이름 · "2025년 9월부터 KEEGO와 함께") + 통계(총 누적 거리·총 러닝) + **그룹 행**(목표: 주간/월간 · 설정: 거리 단위 …).
- 권고: 프로필 헤더 + 누적 통계 + 그룹 행 정렬. 현재 설정/계정/클라우드/백업 기능 전부 보존(최근 ⚙️ 정리와 충돌 주의).

### 러닝 (카운트다운/액티브/요약 — 디자인 `13/14/15`, `RunScreen.js`)
- 디자인: 카운트다운 → 액티브(거리·시간·평균페이스 큰 숫자 + ▶/II/■ 컨트롤) → 요약(저장하기).
- 현재: `RunCountdownScreen` · `RunActiveScreen` · 요약 존재.
- 권고: 액티브 화면의 큰 숫자 레이아웃·컨트롤 버튼 시각 정렬. **GPS/트래킹 로직은 절대 건드리지 말 것**(네이티브·엔진).

### 온보딩 (디자인 `01–04`, `OnboardingScreen.js`)
- 디자인: KEEP GOING 스플래시 + 3 정보 스텝(간단).
- 현재: 6스텝 시네마틱(Welcome→ShoesMatter→Injury→Management→Register→Ready) — **디자인보다 더 완성도 높음**.
- 권고: **현재 시네마틱 유지**(더 나은 부분). 사진 버그는 수정 완료(`0bb416f`).

## 4. 제약 & 검증 방법 (다음 세션 참고)
- **운영 백엔드 시드 금지**: `solelife-backend.onrender.com` 은 공용 운영 DB → POST 거부됨(정상 가드). populated 화면 검증은 **App.tsx 부팅부에 임시 로컬 시드**(globalThis 가드, 커밋 전 제거)로 했음. 운영 무관.
- 에뮬레이터/Metro가 장시간 후 종료될 수 있음 → 재기동: `emulator -avd Pixel_7`, `npx react-native start`, `adb reverse tcp:8081 tcp:8081`, 디버그 앱은 임베디드 번들 없는 새 빌드라 Metro 필수.
- 코드 작업(재구성·tsc·lint·jest)은 에뮬레이터 없이 가능 → 먼저 코드+테스트 green, 마지막에 에뮬레이터 시각 비교.
- 디자인 스크린샷은 1265×748 뷰어 캡처라 **하단이 잘림** → 잘린 부분은 `keego-rn` 코드가 정답.
